# 🔧 كيفية تشغيل التطبيق - إصلاح مشاكل التصميم والبيانات

## المشاكل التي تم إصلاحها:
1. ✅ إعداد NativeWind بشكل صحيح (التصميم لم يكن يعمل)
2. ✅ إضافة metro.config.js
3. ✅ تحديث عنوان API للاتصال بالخادم
4. ✅ إضافة نظام التوجيه التلقائي للمصادقة

## خطوات التشغيل:

### 1️⃣ تأكد من تشغيل الخادم (Backend)
```bash
# يجب أن يكون الخادم يعمل على:
# Server running on port 8080
```

### 2️⃣ تحديث ملف .env
الملف `.env` موجود الآن ومحدث بعنوان IP الخاص بك:
```
EXPO_PUBLIC_API_URL=http://192.168.0.105:8080/api
```

**مهم جداً:**
- ✅ استخدم `192.168.0.105` للهاتف الحقيقي
- ✅ استخدم `10.0.2.2` لمحاكي Android
- ✅ تأكد أن الهاتف والكمبيوتر على نفس شبكة الواي فاي

### 3️⃣ إعادة تشغيل التطبيق
**مهم:** يجب إعادة التشغيل بالكامل بعد التغييرات:

```bash
# أوقف الخادم الحالي (Ctrl+C)
# ثم شغّل من جديد:
npm start -- --clear
```

### 4️⃣ اختبار التطبيق
1. افتح التطبيق على الهاتف/المحاكي
2. يجب أن ترى:
   - ✅ الألوان والتصميم يعمل بشكل صحيح
   - ✅ البيانات تُحمّل من الخادم
   - ✅ صفحة تسجيل الدخول تظهر أولاً

## 🔍 للتحقق من الأخطاء:

### إذا لم تظهر البيانات:
1. افتح Metro bundler terminal
2. ابحث عن رسائل "API Request" و "API Response"
3. إذا رأيت أخطاء مثل "Network Error":
   - تأكد أن الخادم يعمل
   - تأكد أن IP صحيح في `.env`
   - تأكد أن الهاتف على نفس الواي فاي

### إذا لم يظهر التصميم:
1. أعد تشغيل Metro bundler بـ: `npm start -- --clear`
2. تأكد أن ملف `metro.config.js` موجود
3. تأكد أن `global.css` يتم استيراده في `app/_layout.tsx`

## 📱 اختبار سريع:
بعد إعادة التشغيل، يجب أن ترى:
- صفحة Login/Register جميلة بتصميم احترافي
- ألوان البرتقالي والأسود
- البيانات تُحمّل في صفحة Home

## إذا استمرت المشاكل:
قم بإرسال لقطة شاشة من:
1. Terminal حيث يعمل `npm start`
2. الشاشة التي تراها في التطبيق
3. أي رسائل خطأ
